Chapter 4. Converting and Casting

A. Notion Summary
Primitives and Conversion

  • primitive conversion might occur in 3 contexts:

–         assignment,

–         method call, and

–         arithmetic promotion.

  • assigment conversion: happens when you assign a value to a variable of different type.
  • general rules for primitive assignment:
  • a boolean cannot be converted to any other type
  • a non-boolean can be converted to another non-boolean, provided the conversion is a widening conversion
  • a non-boolean cannot be converted to another non-boolean if the conversion is a narrowing conversion
  • if the operand on the right is a literal int, it can be assigned to a byte, short or char without a cast.
  • widening converstions:

–         char to int to long to float to double

–         byte to short to int to long to float to doubl

  • literal values: by default are either an int or a double.
  • primitive converstion: arithmetic promotion
  • unary operators: if the operand is a byte, short or char, it is converted to an int (unless the operator is a ++ or --)
  • binary operators:

–         if one of the operands is a double, the other operand is converted to a double,

–         else if one of the operands is a float, the other operand is converted to a float,

–         else if one of the operands is a long, the other operand is convered to a long,

–         else both are converted to ints.

Primitives and Casting

  • Casts are required when you want to perform a narrowing conversion. Such a conversion is never performed implicitly.
  • Casting to a wider value is always permitted but never required.
  • you can cast any non-boolean type to any other non-boolean type.
  • you cannot cast any type to a boolean or any boolean to another type.

Object Reference Conversion

  • Three kinds of object reference types:

–        class type

–        interface type

–        array type

  • A class or interface can be cast "up" the inheritance heirarchy.
  • A class can be cast to an interface if the class implements that interface.
  • An array can be cast to the class Object or to the interfaces Cloneable or Serializable.
  • An array of object references can be cast to an array of object references of type "up" the inheritance hierarchy.

Object Reference Casting

  • When casting from Oldtype to Newtype:

–         when both are classes, one must be a subtype of the other,

–         when they are both arrays, both must contain reference types (not primitives) and it must be legal to cast Oldtype to Newtype,

–         you can always cast between an interface and a nonfinal object.

–         Runtime compatibility:

–         Newtype must be Oldtype or but a superclass of Oldtype,

–         if Newtype is an interface, Oldtype must implement Newtype.

 

B. Summary based on questions in the text book
1.     In Java primitives and object references can be both converted and cast.
Only casting of object references potentially requires a runtime check. (Runtime checking is a mechanism of a programming language to check for errors at runtime, e.g. arithmetic overflows or invalid type casts. Most times an exception is thrown and/or the program is terminated). Thus, casting of numeric types does not require a runtime check
Objects do not take part in arithmetic operations (Except wrapping classes).

2.    
class A {}
class B extends A {}

//both compile and run ok
B g = new B();
A t= (A) g;
 B h= (B) t;

 // compile ok, but run error
 A t= new A();
 B h= (B) t;
Without the cast to sub-class B you would get a compile time error. The cast tells the compiler that you really mean to do this and the actual type of  t does not get resolved until runtime. Casting down the object hierarchy is a problem, as the compiler cannot be sure what has been implemented in descendent classes. Casting up is not a problem because sub classes will have the features of the base classes. This can feel counter intuitive if you are aware that with primitives casting is allowed for widening operations (ie byte to int).

3.     If s is declared by statement: short s = 55;  then the statement: byte b = s;  will not compile, because converting a short to a byte is a narrowing conversion, which requires an explicit cast. The widening conversions do not need explicit cast.

4.     byte b = 2;  byte b1 = 3;  /*3*/ b = b * b1;
Surprisingly, the code will fail to compile at /*3*/. The two operands, which are originally bytes, are converted to ints before the multiplication. The result of the multiplication is an int, which cannot be assigned to byte b.
void foo(int i) {  };...   char ch = ‘p’; foo(ch);
the char argument ch is widened to type int (a method-call conversion) and passed to the int version of method foo().

5.     Method-call (fun(x)) and assignment (y=x) conversions are governed by the same rules concerning the legal relationships between the old and new types.

6.     Object ob = new Object(); Float x = new Float(3.14f); /*ERROR*/ x = ob;
Changing an Object to a Float is going “down” the inheritance hierarchy tree, so an explicit cast is required.

11.  An array may be converted to another array, provided the element types of the arrays are compatible. This is sometimes, but not always, the case.

12.  The expression !s won’t compile, when s is not boolean.

13.  When a long is cast to a byte, all but the least-significant 8 bits are discarded. The remaining bits might represent a positive number, zero, or a negative number. (In computing, the most significant bit (msb, also called the high-order bit) is the bit position in a binary number having the greatest value. The msb is sometimes referred to as the left-most bit due to the convention in positional notation of writing more significant digits further to the left)

14. The following operators can perform promotion on their operands:  +, -, ~
All numeric operators promote. !, which is strictly boolean, does not promote.
Operator promotion: If an operation is specified with operands of two different types, they are converted to the smallest type that can hold both values. the operator promotion from
> "float" to "double", "int" to "float“ or "int" to "double"...

15.  Method-call conversion and assignment conversion are governed by the same rules.

16.  The following line of code is valid?
int x = 9; byte b = x;
False
Due to the rules of widening conversions, the integer value of x cannot be automatically converted to a byte. The assignment of the variable x to the variable b would require an explicit cast. This cast could result in a loss of data, though.
