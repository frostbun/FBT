Chapter 6. Objects and Classes

A. Notion Summary
Coupling and Cohesion

  • coupling: the degree of an object's reliance on knowledge of the internals of another entity's implementation.
  • cohesion: the degree to which a class or method resists being broken down into smaller pieces.

Overloading and Overriding

  • overloading: re-using a method name in a subclass but with different arguments.
  • overriding: re-using a method name in a subclass with identical arguments.

–         Return type may be a subclass of the one in the parent.

–         Methods marked "final" may not be overridden.

–         covariant return type: A covariant return type of an overriding method is a subclass of the return type of the superclass verion.

–         the accessibility of an overriding method must not be less than that of the method it overrides.

–         an overriding method method may only throw checked exception types that are the same as or subtypes of the original method.

Variable-Length Argument Lists

  • void zyz (String ... stringies)

Constructors and Subclassing

  • default constructor: a public no-args constructor automatically generated if no constructor is coded. It invokes the superclass but does nothing else.
  • super(): invokes the superclass' constructor.
  • you can overload, but not override, constructors.
  • a method can invoke a constructor by this(correct arg), or its super class by super(correct args)

Inner Classes

  • you can declare an inner class inside any block of parens
  • The enclosing class name become part of the fully qualified name of the inner class. In code that is independent of the class loader, the dotted notation is used. In code that is "filename aware", the class name and the inner class name are separated by a $.
  • it is illegal for a package and a class to have the same name.
  • an inner class has access to the outer class members.
  • An inner class cannot be declared in a static method of an outer class without an instance of the outer class existing. The new operator must build them both:
  • OuterOne.InnerOne i = new OuterOne().new InnerOne();

which is equivalent to:

OuterOne o = new OuterOne();
OuterOne.InnerOne i = o.new InnerOne();

  • In a non static member, the "this" reference is assumed:
  • InnerOne i = new InnnerOne(); is ok.

Member Inner Classes

  • an inner class defined in the class scope.
  • access modifiers apply.
  • static member inner class: has no this referencing the enclosing instance. It can reference class (static) members of the enclosing instance, but not instance members. You can create an instance of a static inner class without an instance of the outer class.

InnerOne i = new OuterOne.InnerOne();

  • A static inner member class is just another top-level class with a modified naming sheme. It can even be used as an extention to packaging.

Classes Defined Inside Methods

  • anything declared in a method is local to the method
  • they cannot be marked static or with any access modifier
  • they have access to the method's automatic variables.

Accessing Method Variables

  • Any method variable – local or formal - can be access by methods within an inner class provided that the variable(s) are marked "final".
  • An object created can outlast the method invocation.

Annonymous Classes

  • a class with no name.
  • can extend a single class xor implement a single interface.
  • the definition, construction, and first use all occur in the same place.
  • anonymous classes are unique to method scope
  • it cannot have constructors.
  • it can be constructed with arguments and with an initializer.

Contruction and Initialization of Anonymous Inner Classes

  • the code structure:

new Xxxx() {/* class body */}

  • assignment:

Xxxx anXxxx = new Xxxx(){/* class body */}

  • actual parameter:

someMethod (new Xxxx(){/* class body */});

  • initializing a parent's constructor:

new Xxxx(actual_parameter) {/* class body */}

Contracts and Naming Conventions
equals()

  • uses the == operator
  • any object should equal itself
  • if x equals y, then y equals x
  • if x equals y and y equals z, then x must equal z

hash code

  • it returns an int that represents the uniqueness of an object
  • if 2 objects are equal according to the equals operator, the must have the same hash code
  • it is ok for 2 unequal objects to have the same hash code.

Enums

  • they can be declared anywhere a class is declared
  • inherit from Object
  • can be converted and cast like any object extending Object
  • may have main() methods
  • may not be instantiated
  • may not extend anything and may not be extended
  • may be arguments in switch statements (unlike classes)
  • have built in name() and toString() methods

Interface

  • They can contain final variables and function prototypes only. Members of an interface are public by default. Variables are final by default.
  • A class implementing an interface must overidde all methods in the interface.
  • An interface can be used to create an anonymous object.
  • An interface object can be passed to the function as a parameter.

15. Which restriction should be applied to anonymous inner classes?
An anonymous inner class must appear inside a block of code. There are no restrictions preventing an anonymous inner class from accessing the enclosing class’ non-final data or methods or calling the enclosing class’ synchronized methods.
