Arithmetic Operators
+

  • For a + expression with two operands of primitive numeric type, the result:

–         is of a primitive numeric type.

–         is at least int, because of normal promotions.

–         is a a type as wide as the wider of the two operands.

–         has a value calculated by promoting the operands to the result type and then performing the addition using that type. This might result in overflow or loss of precision.

  • For a + expression with any operand that is not of primitive numeric type,

–         at least one operand must be a String object or literal; otherwise the expression is illegal.

–         any remaining non-String operands are converted to String, and the result of the expression is the concatenation of all operands.

–         To convert an operand of some object type to a String, the conversion is performed by invoking the toString() method of that object.

–         To convert an operand of a primitive type to a String, the conversion is performed by a static method in a container class, such as Integer.toString().

The Comparison Operators
The comparison operators return a boolean result: true or false.
The ordinal comparisons test the relative values of numeric operands.

  • < > <= >=

–         not applicable to any non-numeric type

–         char is a numeric type.

–         the narrower type is promoted to the wider type before the comparison.

Object-type comparisons determine whether the run-time type of an object of a particular type or a subclass of that particular type.

  • the right operand must be a class, interface or array type.
  • a compiler error results if the left operand cannot be cast to the right operand.

Equality comparisons test whether two values are the same and may be applied to values of non-numeric types.
These operators work on dissimilar numeric types and char, but not booleans or class types.
Equality comparison operators: == and !=

  • return a boolean
  • when applied to variables of object type, they test reference addresses, not object values.
  • to compare object values, use the equals() method.

Bitwise Operators

  • Applicable to integral and boolean types.
  • & : and
  • ^ : xor
  • | : or

The Short-Circuit Logical Operators

  • && and || are operations on boolean types only.

The Conditional Operator: a ternary operator

  • ?=
  • if a, b, c are ints and x is a boolean expression:

if (x) {
a = b;
} else {
a = c;
}

is identical to a = x ? b : c;

  • b and c shold be type compatible
  • x should be boolean
  • b and c should be assignment compatible with a
  • the value assigned to a will be b if x is true or will be c if x is false

Assignment Operators

  • =, +=, -=, *=, /=

B. Summary based on questions in the text book
1.     >> is the signed right-shift operator, so the result has the same sign as the left-hand operand. Right-shifting by n bit positions is equivalent to dividing by 2-to-the-n, so the result of  -50>>1  is -50 / 2, which is -25.
int x = 25>>3; int y = -25>>3; =>  x = 3, y = -4
2.     int x;  y  =  x >>> k;
       x>>>k is an int (thus the statements byte y = x>>>k; or short  y = x>>>k; do not compile). Bits of  x   are shifted  right by distance  k%32; fills with 0 bits on the left side, so 0s are shifted into the most significant positions; any two’s-complement number with a 0 in the most significant position is positive. If k < 32 the result is always positive. If  k = 32   y is the same as x.
3.     string = string + number is legal, string = number or string = number + string is illegal
4.     If  k and h are positive integer  then  -k %h = - (k%h), k/-h = k%h, thus -k%-h = -k%h. e.g. 7%4 = 7%-4 = 3, -7%4 = -3.
5.     The XOR operator produces a 1 bit in any position where the operand bits are different. So
00001010 ^ 00001111 is 00000101, which is 5.
6.     int x = 5;  System.out.println((x > 5) ? 99.99 : 9);
The output is 9.0. In this code, the optional result values for the conditional operator, 99.99 (a double) and 9 (an int), are of different types. The result type of a conditional operator must be fully determined at compile time, and in this case the type chosen, using the rules of promotion for binary operands, is double. Because the result is a double, the output value is printed in a floating point format. If the two possible argument types had been entirely incompatible—for example, (x > 4) ?"Hello" : 9—then the compiler would have issued an error at that line.
7.     When a byte is added to a char, the type of the result is int. When a short is added to a float, the type of the result is float
8.     Among Java operators only non-floating division can throw an ArithmeticException. Thus
int k = 5, h = 0; System.out.println(k/h);  // throw an ArithmeticException
float k = 5, h = 0;  System.out.println(k/h); // do not throw an ArithmeticException
9.     The instanceof operator generates a boolean value. Only a reference may be the left-hand operand of instanceof operator. Classes, interfaces, arrays of classes, and arrays of interfaces may be the right-hand operands of instanceof. operator.
String str = new String("ABC"); // If str is not intitialized then there will be compile error
boolean t = str  instanceof  String; // t get true value
10.  try {int x = 5;} catch(Exception e) {System.out.println(x);} //  Error, because  x is  undefined in catch block
11.  String  x = "ABC"; String y = new String("ABC"); String z = y.intern(); // z is placed in pool after intern() is called. The result  of  x == z  is true.
12.  What is the value of x after the following operation is performed? x = 23 % 4;
3
        The modulo (%) operator returns the leftover value after a division operation. In the given example, 23 / 4 = 5, with 3 remaining after the division. Therefore, the answer is 3.
